Fehlersuche und Fehlerbeseitigung

Nachfolgend sind einige der häufigsten Probleme und Fragen zum Betrieb von Tor aufgeführt. Falls Sie zu Ihrem Problem nicht fündig werden, finden Sie auf unserer Webseite unter www.vidalia-project.net weitere Hilfe und Informationen.

Ich kann Tor nicht starten

Ein häufiger Grund, weshalb Vidalia Tor nicht starten konnte, ist die Angabe eines falschen Tor-Verzeichnisses. Sie können Vidalia das richtige Tor-Verzeichnis mitteilen, indem Sie die Option Pfad zur Tor-Anwendung in den Allgemeinen Einstellungen korrigieren.

Ein anderer möglicher Grund ist, dass bereits ein Tor-Prozess läuft. Überprüfen Sie die Liste der laufenden Prozesse und beenden Sie den laufenden Tor-Prozess, falls Sie einen finden. Versuchen Sie anschließend Tor erneut zu starten.

Falls dieses Vorgehen nicht hilft, überprüfen Sie das Nachrichtenprotokoll auf Fehler während des Starts von Tor.

Vidalia kann keine Verbindung zu Tor herstellen

Vidalia steuert Tor durch Kommunikation über den Kontrollport von Tor.

Der häufigste Grund, dass Vidalia keine Verbindung zu Tor herstellen konnte, ist dass Tor nach dem Start einen Fehler festgestellt hat und sofort beendet wurde. Sie können das Nachrichtenprotokoll anschauen und herausfinden, ob Tor beim Start Fehler gemeldet hat.

Falls Tor auf einem anderen Port auf Verbindungen wartet, als es Vidalia erwartet, kann Vidalia keine Verbindung zu Tor herstellen. Diese Einstellung muss nur selten geändert werden, bspw. müssen Sie einen anderen Port angeben, falls ein anderer Dienst auf Ihrem Rechner ebenfalls diesen Port benutzt. Diese Einstellung können Sie in den Fortgeschrittenen Einstellungen von Vidalia vornehmen.

Tor wurden unerwartet beendet

Falls Tor sofort nach dem Start beendet wird, läuft wahrscheinlich bereits ein anderer Tor-Prozess. Schauen Sie im Nachrichtenprotokoll nach, ob eine der letzten Meldungen gelb hervorgehoben ist und eine Nachricht ähnlich der folgenden enthält: 

connection_create_listener(): Could not bind to 127.0.0.1:9050: Address already in use. 
Is Tor already running?

Falls Sie eine solche Fehlermeldung finden, müssen Sie den anderen Tor-Prozess beenden, bevor Sie einen neuen mit Vidalia starten. Falls Sie Windows benutzen, können Sie nach tor.exe im Taskmanager suchen und diesen Prozess beenden. Auf anderen Betriebssystemen hilft Ihnen der Befehl ps den anderen Tor-Prozess zu finden.

Falls Tor bereits länger als einige Sekunden lief, bevor es beendet wurde, sollten Sie das Nachrichtenprotokoll auf Informationen durchsuchen. Fehler die zum Beenden von Tor geführt haben, werden rot oder gelb hervorgehoben.

Vidalia kann Tor nicht beenden

Falls Vidalia Tor nicht beenden konnte, sollten Sie das Nachrichtenprotokoll auf Fehler durchsuchen, die beim Beenden von Tor aufgetreten sind.
